# Phase 4 TODO List – Advanced Features & Production Readiness

Phase 4 Goals: Add advanced utilities, improve developer experience, and harden the library for production use in the Cobranza App ecosystem.

Definition of Done:

- All advanced features are implemented and covered by tests.
- Documentation is updated with new capabilities and security guidance.
- The library remains lightweight, secure, and easy to use in other microservices.

## TODO Tasks – Phase 4

### Task 1: Advanced Cryptographic Features [DONE]

- [x] Implement **bulk object operations** in `src/crypto.service.ts`:
  - `encryptObject<T>(obj: T, fieldMap: Record<keyof T, EncryptionKey>): T` – Encrypts multiple fields in one object according to a mapping.
  - `decryptObject<T>(obj: T, fieldMap: Record<keyof T, EncryptionKey>): T` – Decrypts specified fields.
- [x] Add `reEncrypt(encrypted: EncryptedValue, targetKeyName?: EncryptionKey): EncryptedValue` – Re-encrypts data with a (possibly) different key while updating version.
- [x] Implement a simple **decryption cache** utility in `src/utils/decryption-cache.ts`:
  - In-memory cache with configurable TTL.
  - Provide `withCache()` wrapper method on `SecureCrypto`.

### Task 2: Observability & Auditing

- [ ] Add optional event hooks to `SecureCrypto`:
  - Allow passing an `AuditLogger` interface in `CryptoConfig` with methods like `onEncrypt(keyName: string, version: number)` and `onDecrypt(keyName: string, version: number)`.
  - Ensure no sensitive data (plaintext or full ciphertext) is ever logged.

### Task 3: Security Hardening

- [ ] Strengthen input validation in all public methods (check for empty values, invalid Base64, wrong key lengths, etc.).
- [ ] Ensure proper buffer cleanup / zeroing where feasible after sensitive operations.
- [ ] Add a dedicated security documentation section in new /docs file (and link it in README.md) covering:
  - Recommended key storage practices.
  - Rotation procedure.
  - Common pitfalls to avoid.
