# Plan: Auto-Allow Frequent-Touch Files Without Permission Prompts

**Date**: 2026-05-27

## Problem

Every time an AI agent modifies frequently-touched project files during the Critical Workflow, the user gets a permission prompt. Since these files are modified often, repeated prompts are painful.

**Affected files/directories**:
- `.kilo/state.json` — updated at every sub-step
- `.agent/project-info/` — context.md, tech.md, architecture.md, etc.
- `.agent/todos/` — task tracking files
- `.agent/project-structure.md` — project structure map
- `.kilo/plans/` — implementation plan files
- `src/` — all source code
- `docs/` — documentation
- `.gitignore` — project exclusions
- `.kilocodeignore` — indexing exclusions
- `CHANGELOG.md` — changelog
- `LICENSE` — license
- `README.md` — readme

## Root Cause

The project's `.kilo/kilo.jsonc` has no `permission` block. Without an explicit `allow` rule, the `edit` tool falls back to `ask` (prompt) for all file writes.

## Solution

Add a `permission` block to `.kilo/kilo.jsonc` that explicitly `allow`s writes to all the files/directories listed above. All other files remain on `ask`.

### Step 1: Update `.kilo/kilo.jsonc`

**Current content**:
```jsonc
{
  "$schema": "https://app.kilo.ai/config.json",
  "agent": {
    "plan": {
      "enabled": true
    }
  },
  "instructions": [
    ".kilo/rules/**/*.md",
    ".kilo/commands/**/*.md"
  ]
}
```

**New content** (adds `permission` object):
```jsonc
{
  "$schema": "https://app.kilo.ai/config.json",
  "agent": {
    "plan": {
      "enabled": true
    }
  },
  "instructions": [
    ".kilo/rules/**/*.md",
    ".kilo/commands/**/*.md"
  ],
  "permission": {
    "edit": {
      ".kilo/state.json": "allow",
      "src/**": "allow",
      ".agent/project-info/**": "allow",
      ".agent/todos/**": "allow",
      ".agent/project-structure.md": "allow",
      ".kilo/plans/**": "allow",
      "docs/**": "allow",
      ".gitignore": "allow",
      ".kilocodeignore": "allow",
      "CHANGELOG.md": "allow",
      "LICENSE": "allow",
      "README.md": "allow",
      "*": "ask"
    }
  }
}
```

**Rationale**:
- Each pattern targets exactly the files/directories requested
- `.kilo/state.json` is a single-file match (not `.kilo/**`) — only the state file, not other `.kilo/` config files
- `*`: `"ask"` fallback preserves the existing prompt behavior for everything else (e.g., `.kilo/kilo.jsonc`, `.kilo/agents/`, root config files)
- Permission rules are evaluated top-to-bottom, first match wins — the `"*"` catch-all at the bottom ensures all unmatched files still prompt

|------|--------|
| `.kilo/kilo.jsonc` | Add `permission.edit` block with allow rules |
