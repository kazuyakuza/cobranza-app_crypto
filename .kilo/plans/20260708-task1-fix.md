## Issues Found

### Issue 1: `package.json` lint script deviates from plan

**Plan specified** (`.kilo/plans/20260708-task1-config.md` §1.3):

```json
"lint": "eslint \"src/**/*.ts\" \"tests/**/*.ts\""
```

**Current value** (`package.json` line 30):

```json
"lint": "eslint --no-error-on-unmatched-pattern \"src/**/*.ts\" \"tests/**/*.ts\""
```

**Impact**: Low. The flag prevents ESLint from failing when `tests/` does not yet exist (it will be created in Task 4). It does not change lint behavior once test files exist.

**Fix options**:

1. **Preferred — keep the flag and document the deviation**
   - Rationale: `tests/` is intentionally empty at this stage (Task 4 creates the first test files). Removing the flag now would make `npm run lint` fail until Task 4 is completed, violating the plan's own verification step (§2.5).
   - Action: Update `.kilo/plans/20260708-task1-config.md` §1.3 to reflect the actual script and explain why `--no-error-on-unmatched-pattern` was added.

2. **Alternative — align exactly with the original plan**
   - Action: Remove `--no-error-on-unmatched-pattern` from the `lint` script in `package.json`.
   - Consequence: `npm run lint` will fail with "No files matching the pattern \"tests/**/*.ts\" were found" until Task 4 creates `tests/`. The Step 4.2 verification will no longer pass without also creating at least one file under `tests/`.

**Recommended fix**: Option 1. The deviation is justified by the current absence of `tests/` and will become irrelevant once Task 4 is implemented.
