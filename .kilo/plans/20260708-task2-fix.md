### 4. `src/utils.ts` — Minor documentation inconsistency, optional fix

- Helper functions present: `base64ToBuffer`, `bufferToBase64`, `generateIv`, `concatBuffers`, `constantTimeCompare`.
- `constantTimeCompare` implementation deviates slightly from the plan:
  - Plan expected: `return timingSafeEqual(aBuffer, bBuffer) === EQUAL_LENGTH_MATCH;`
  - Implementation: `return timingSafeEqual(aBuffer, bBuffer);`
  - **No code change required**: Node.js `crypto.timingSafeEqual` returns a `boolean`, so the implementation is correct and the plan's `EQUAL_LENGTH_MATCH` comparison would have been wrong.
- `base64ToBuffer` JSDoc states `@throws {Error} when value is empty or not valid base64`.
  - Implementation only throws when `value` is empty/falsy.
  - `Buffer.from(value, 'base64')` silently ignores invalid base64 characters instead of throwing.
  - **Recommended optional fix**: update JSDoc to `@throws {Error} when value is empty` so the documentation matches runtime behavior.
- File length and method lengths are within limits.
