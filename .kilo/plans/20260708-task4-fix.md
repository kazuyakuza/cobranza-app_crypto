# Fix Plan — Task 4: Testing Infrastructure (Phase 1)

## Issue Found

`.eslintrc.json` was changed in the implementation commit to add `"tests/"` to `ignorePatterns`. This silently disables ESLint for `tests/crypto.service.spec.ts`, contradicting the implementation plan's Step 6 instruction: "If eslint flags the spec or testing files, fix per eslint guidance (do not disable rules silently)."

### Why the change was made

`tsconfig.json` excludes `tests/` and `**/*.spec.ts`. With `parserOptions.project: "./tsconfig.json"`, ESLint cannot parse files outside the TSConfig include/exclude set, so the implementation avoided the parse error by ignoring the whole `tests/` directory.

### Why it is a deviation

The plan expects lint to run on the new spec file and for any issues to be fixed properly. Ignoring the directory hides potential lint problems in test code instead of resolving them.

## Proposed Fix

1. **Create `tsconfig.eslint.json`**
   - Extend `./tsconfig.json`.
   - Override `include` to `["src/**/*.ts", "tests/**/*.ts"]`.
   - Override `exclude` to `["dist", "node_modules"]` so the base `exclude` does not remove `tests/**/*.spec.ts`.

2. **Update `.eslintrc.json`**
   - Change `parserOptions.project` from `"./tsconfig.json"` to `"./tsconfig.eslint.json"`.
   - Remove `"tests/"` from `ignorePatterns`.

3. **Run lint and fix any new errors**
   - `npm run lint`
   - Address any errors or warnings in `tests/crypto.service.spec.ts` or `src/testing/*.ts` without disabling rules.

4. **Re-run verification suite**
   - `npm run build` — must succeed and emit `dist/testing/*`.
   - `npm run test` — must show 12 passing tests.
   - `npm run lint` — must pass with `tests/` now included.

5. **Commit**
   - Stage `tsconfig.eslint.json` and `.eslintrc.json`.
   - Commit with a meaningful message, e.g.:
     ```
     fix(crypto): lint tests directory instead of ignoring it
     
     - Add tsconfig.eslint.json covering src and tests.
     - Point ESLint parserOptions.project to tsconfig.eslint.json.
     - Remove tests/ from eslint ignorePatterns.
     ```

## Verification Performed During Review
